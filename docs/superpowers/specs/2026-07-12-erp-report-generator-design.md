# مولد تقارير ERP — وثيقة التصميم (Spec)

**التاريخ:** 2026-07-12
**الحالة:** معتمد
**اللغة:** PHP 8.3 — واجهات عربية RTL احترافية

## 1. الهدف والنطاق

مولد تقارير احترافي لشركة ERP، بجودة عرض (Portfolio) وبيانات تجريبية واقعية جاهزة.
يوفّر **٣ تقارير جاهزة (Templates)** + لوحة تحكم:

- **المبيعات** — فواتير، إجماليات، أفضل المنتجات، رسم بياني حسب الفترة.
- **المخزون** — الكميات، حد الطلب، الأصناف الناقصة، قيمة المخزون.
- **المالية** — الإيرادات مقابل المصروفات، صافي الربح، تدفق نقدي شهري.

**خارج النطاق (YAGNI):** تقرير الموظفين/HR، تسجيل الدخول/المصادقة، إدارة المستخدمين، تعدد اللغات، الطباعة بمكتبة PDF ثقيلة.

## 2. المقاربة التقنية

PHP نقي بهيكل **MVC مُبسّط** + قاعدة بيانات **SQLite** مدمجة. بدون أطر ثقيلة وبدون Composer.
التشغيل عبر خادم PHP المدمج: `php -S localhost:8000 -t public`.

**سبب الاختيار:** احترافي، خفيف، ذاتي الاحتواء، سهل الاختبار الآلي بـ Playwright، ولا يتطلب إعداد قاعدة بيانات خارجية.

## 4. طبقة البيانات (SQLite)

**الجداول:**
- `products` — id, name, category, price, cost, stock_qty, reorder_level.
- `sales_invoices` — id, invoice_no, customer_name, invoice_date, total.
- `sales_items` — id, invoice_id, product_id, qty, unit_price, line_total.
- `transactions` — id, type (income|expense), category, amount, txn_date, description.

**البذور (seed.php):** ~٣٠ منتج عبر تصنيفات، ~٢٠٠ فاتورة موزّعة على ١٢ شهر مع بنودها، ~١٥٠ حركة مالية (إيرادات ومصروفات). أرقام واقعية متسقة (إجمالي الفاتورة = مجموع بنودها).

## 5. التقارير والميزات

| التقرير | المؤشرات (KPIs) | الجدول | الرسم | الفلاتر |
|--------|------------------|--------|-------|---------|
| لوحة التحكم | إجمالي المبيعات، صافي الربح، قيمة المخزون، أصناف ناقصة | آخر الفواتير | مبيعات شهرية (خطي) + إيراد/مصروف (أعمدة) | — |
| المبيعات | إجمالي المبيعات، عدد الفواتير، متوسط الفاتورة | الفواتير + أفضل المنتجات | مبيعات حسب الشهر | من/إلى تاريخ |
| المخزون | إجمالي الأصناف، قيمة المخزون، أصناف ناقصة | المنتجات + تمييز الناقص | قيمة المخزون حسب التصنيف | التصنيف |
| المالية | الإيرادات، المصروفات، صافي الربح | حركات مالية | إيراد مقابل مصروف شهري | من/إلى تاريخ |

## 6. الواجهة (UX)

- عربي RTL بالكامل، خط عربي (Cairo/Tajawal عبر CDN).
- نظام تصميم: Sidebar تنقّل، Header، بطاقات KPI، جداول أنيقة، لوحة ألوان متناسقة (light).
- متجاوب (Grid/Flex)، حالات فارغة واضحة.
- **نمط طباعة (`@media print`):** يخفي التنقّل ويُخرج التقرير نظيفاً للحفظ كـ PDF.

## 7. التصدير

- **CSV:** توليد أصلي في PHP عبر `ExportController` (header: Content-Type + Content-Disposition).
- **Excel:** ملف `.xls` بصيغة HTML-table (يفتحه Excel أصلاً) — بدون تبعيات.
- **PDF:** عبر نمط الطباعة CSS + `window.print()` (المتصفح → حفظ كـ PDF).

## 8. الاختبار

**يدوي:** تشغيل الخادم وتصفّح كل صفحة والفلاتر والتصدير.

**آلي (UI Automation — Playwright MCP):**
- التنقّل لكل صفحة والتحقق من رمز الاستجابة والعنوان.
- التحقق من ظهور بطاقات KPI والجداول (عدد صفوف > 0) وعناصر الرسوم (canvas).
- اختبار الفلاتر (تطبيق تاريخ/تصنيف وتغيّر النتائج).
- اختبار روابط التصدير (استجابة صحيحة / تنزيل).
- التقاط لقطات شاشة لكل صفحة تُرفق في تقرير المشروع.

## 9. سير عمل GitHub (رفع أول بأول)

1. تثبيت `gh CLI` ثم `gh auth login` (متصفح).
2. `gh repo create` لإنشاء الريبو (يقوم به كلود — الشرط #1).
3. **دفع تدريجي (commit + push) بعد كل مرحلة:**
   - المرحلة ٠: الهيكل + README + الـ spec.
   - المرحلة ١: Core (Router/Database/Controller).
   - المرحلة ٢: schema + seed + قاعدة البيانات.
   - المرحلة ٣: تقرير المبيعات.
   - المرحلة ٤: تقرير المخزون.
   - المرحلة ٥: تقرير المالية + لوحة التحكم.
   - المرحلة ٦: التصدير + نمط الطباعة.
   - المرحلة ٧: اختبارات Playwright + اللقطات.
   - المرحلة ٨: تقرير محتويات المشروع.
   - `.gitignore` يستبعد `database/erp.sqlite` والملفات المؤقتة.

## 10. التسليم النهائي

`docs/PROJECT_REPORT.md` — تقرير محتويات المشروع: البنية، وصف المكوّنات، كيفية التشغيل، لقطات الواجهات، نتائج الاختبار الآلي، ورابط الريبو.

## 11. معايير النجاح

- الخادم يعمل عبر `php -S` بلا أخطاء.
- التقارير الثلاثة + لوحة التحكم تعرض بيانات واقعية وتعمل فلاترها.
- التصدير (CSV/Excel) يعمل، والطباعة تُخرج PDF نظيف.
- اختبارات Playwright تمرّ على كل الصفحات.
- الريبو منشأ على GitHub بواسطة كلود، والمهام مرفوعة أول بأول.
- تقرير المشروع مكتمل ومرفوع.
